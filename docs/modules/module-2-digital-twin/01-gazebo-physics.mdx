# Gazebo Physics: Gravity, Collisions

## Introduction

Gazebo is a powerful 3D simulation environment that provides accurate physics simulation for robotics applications. This chapter explores how Gazebo simulates fundamental physics concepts like gravity and collisions that are crucial for humanoid robot simulation.

## Gazebo Architecture

Gazebo consists of several key components:
- **Physics Engine**: Provides realistic physics simulation (ODE, Bullet, SimBody)
- **Sensor Simulation**: Models various sensors with realistic noise and dynamics
- **Rendering Engine**: Provides visual rendering for simulation environment
- **Plugin System**: Allows customization of robot and world behavior

## Physics Simulation Fundamentals

### Gravity Simulation

Gravity is a fundamental force in Gazebo that affects all objects in the simulation. The gravity vector is typically set to (0, 0, -9.8) to simulate Earth's gravity:

```xml
<world>
  <gravity>0 0 -9.8</gravity>
  <!-- World content -->
</world>
```

For humanoid robots, proper gravity simulation is essential for:
- Walking and balance algorithms
- Fall detection and recovery
- Proper weight distribution in joints
- Realistic interaction with the environment

### Collision Detection

Gazebo uses collision detection to determine when objects interact with each other. There are two main aspects:

1. **Collision Geometry**: Defines the shape used for collision detection
2. **Surface Properties**: Defines how objects behave when they collide

```xml
<link name="link_name">
  <collision>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry>
      <box size="1 1 1"/>
    </geometry>
    <surface>
      <friction>
        <ode>
          <mu>1.0</mu>
          <mu2>1.0</mu2>
        </ode>
      </friction>
      <bounce>
        <restitution_coefficient>0.1</restitution_coefficient>
        <threshold>100000</threshold>
      </bounce>
    </surface>
  </collision>
</link>
```

## Physics Engines in Gazebo

### ODE (Open Dynamics Engine)

ODE is the default physics engine in Gazebo and is suitable for most humanoid robotics applications:
- Good balance between accuracy and performance
- Supports complex joint types
- Well-tested for robotic applications

### Bullet Physics

Bullet offers more advanced features:
- Better handling of complex contact scenarios
- More accurate friction models
- Support for soft body dynamics

### SimBody

SimBody provides high-fidelity simulation:
- Best accuracy for complex multi-body systems
- Advanced constraint handling
- Used for biomechanical simulations

## Collision Properties for Humanoid Robots

### Friction Parameters

For humanoid robots, friction properties are crucial for stable walking:

```xml
<surface>
  <friction>
    <ode>
      <mu>0.8</mu>    <!-- Primary friction coefficient -->
      <mu2>0.8</mu2>  <!-- Secondary friction coefficient -->
      <fdir1>0 0 1</fdir1>  <!-- Friction direction -->
    </ode>
  </friction>
</surface>
```

### Contact Parameters

Contact parameters define how surfaces interact during collisions:

```xml
<surface>
  <contact>
    <ode>
      <soft_cfm>0</soft_cfm>
      <soft_erp>0.2</soft_erp>
      <kp>1000000.0</kp>  <!-- Contact stiffness -->
      <kd>1.0</kd>        <!-- Contact damping -->
      <max_vel>100.0</max_vel>
      <min_depth>0.001</min_depth>
    </ode>
  </contact>
</surface>
```

## World Definition

A complete Gazebo world includes physics parameters:

```xml
<sdf version="1.6">
  <world name="humanoid_world">
    <!-- Physics parameters -->
    <physics type="ode">
      <max_step_size>0.001</max_step_size>
      <real_time_factor>1.0</real_time_factor>
      <real_time_update_rate>1000.0</real_time_update_rate>
      <gravity>0 0 -9.8</gravity>
    </physics>

    <!-- Models and plugins -->
    <include>
      <uri>model://ground_plane</uri>
    </include>

    <include>
      <uri>model://sun</uri>
    </include>
  </world>
</sdf>
```

## Humanoid-Specific Physics Considerations

### Center of Mass

Proper center of mass calculation is essential for humanoid stability:

```xml
<link name="torso">
  <inertial>
    <mass>5.0</mass>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    <inertia ixx="0.5" ixy="0.0" ixz="0.0"
             iyy="0.5" iyz="0.0" izz="0.3"/>
  </inertial>
</link>
```

### Joint Dynamics

For humanoid joints, proper dynamics parameters ensure realistic movement:

```xml
<joint name="hip_joint" type="revolute">
  <parent>torso</parent>
  <child>thigh</child>
  <axis>
    <xyz>0 0 1</xyz>
    <limit>
      <lower>-1.57</lower>
      <upper>1.57</upper>
      <effort>100</effort>
      <velocity>1</velocity>
    </limit>
    <dynamics>
      <damping>0.1</damping>
      <friction>0.0</friction>
    </dynamics>
  </axis>
</joint>
```

## Simulation Accuracy vs Performance

Gazebo offers trade-offs between simulation accuracy and performance:

### Accuracy Settings
- Smaller step size (e.g., 0.001s)
- Higher update rate
- More precise collision detection

### Performance Settings
- Larger step size (e.g., 0.01s)
- Lower update rate
- Simplified collision geometry

## Gazebo Plugins for Physics

Gazebo supports various plugins to enhance physics simulation:

### Joint Control Plugin

```xml
<plugin name="joint_controller" filename="libgazebo_ros_joint_state_publisher.so">
  <robotNamespace>/humanoid_robot</robotNamespace>
  <jointName>hip_joint</jointName>
</plugin>
```

### Physics State Plugin

```xml
<plugin name="physics_state" filename="libgazebo_ros_p3d.so">
  <alwaysOn>true</alwaysOn>
  <updateRate>100.0</updateRate>
  <bodyName>base_link</bodyName>
  <topicName>odom</topicName>
</plugin>
```

## Best Practices for Humanoid Simulation

1. **Start Simple**: Begin with basic physics and gradually add complexity
2. **Validate Parameters**: Test physics parameters with real robot data when available
3. **Monitor Performance**: Balance accuracy with simulation speed
4. **Use Proper Scaling**: Ensure all units are consistent (SI units recommended)
5. **Test Extremes**: Validate behavior under extreme conditions (fast movements, impacts)

## Troubleshooting Common Physics Issues

### Unstable Simulation
- Reduce step size
- Increase damping values
- Check mass and inertia parameters

### Penetration Issues
- Increase contact stiffness (kp)
- Reduce step size
- Improve collision geometry

### Joint Limit Problems
- Verify joint limits in URDF/SDF
- Check controller parameters
- Ensure proper control loop frequency
